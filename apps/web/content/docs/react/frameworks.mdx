---
title: Framework adapters
description: Mount the gateway in Next.js, Hono, or Express. Each binding is a thin marshalling shim onto the one framework-agnostic core, router.handle(req).
---

[`createFilesRouter`](/react/gateway) returns a framework-agnostic core — `handle(req: Request): Promise<Response>`. A framework adapter is just the shim that turns that framework's request/response into a Web `Request`/`Response`. Each one exports a `createRouteHandler`, so the only thing that changes between frameworks is the mount call.

Build the router once (it composes your adapter, `authorize`, plugins, and prefix), then hand it to the binding for your framework:

```ts
import { createFiles } from "files-sdk";
import { s3 } from "files-sdk/s3";
import { createFilesRouter } from "files-sdk/api";

export const router = createFilesRouter({
  files: createFiles({ adapter: s3({ bucket: "uploads" }) }),
  allowedOrigins: ["https://app.example.com"],
  authorize: async ({ operation, req }) => {
    /* throw to deny, or return a per-user constraint — see /react/authorization */
  },
});
```

## Next.js

`files-sdk/next` targets the App Router. `createRouteHandler` returns `{ GET, POST, PUT }` — `GET` serves downloads, `POST` the JSON verbs, `PUT` the upload byte path. The handler is Web-native (`Request`/`Response`, `crypto.subtle`, `ReadableStream`), so the same route runs on Node **and** the Edge runtime.

```ts title="app/api/files/route.ts" lineNumbers
import { createRouteHandler } from "files-sdk/next";
import { router } from "./router";

export const { GET, POST, PUT } = createRouteHandler(router);
```

## Hono

`files-sdk/hono` is Web-native too, so it runs anywhere Hono does — Cloudflare Workers, Bun, Deno, and Node. `createRouteHandler` returns a single handler; mount it with `app.all` so all three methods reach the gateway (it dispatches on the method internally, and addresses everything by query string, so one fixed path is enough).

```ts title="server.ts" lineNumbers
import { Hono } from "hono";
import { createRouteHandler } from "files-sdk/hono";
import { router } from "./router";

const app = new Hono();
app.all("/api/files", createRouteHandler(router));

export default app;
```

## Express

`files-sdk/express` bridges a Node `IncomingMessage`/`ServerResponse` to the Web `Request`/`Response` the gateway speaks (`Readable.toWeb`/`fromWeb` + `pipeline`). It's typed against `node:http`, so it also works with Connect and a raw `http.createServer`. A client disconnect is wired through to abort the upstream read on a proxied download.

```ts title="server.ts" lineNumbers
import express from "express";
import { createRouteHandler } from "files-sdk/express";
import { router } from "./router";

const app = express();

// Mount the gateway BEFORE any body parser (see below).
app.all("/api/files", createRouteHandler(router));

// Body parsers for the rest of your app come after.
app.use(express.json());

app.listen(3000);
```

<Callout type="warn">
  A body parser (`express.json()`, `express.urlencoded()`, …) consumes the
  request stream. The gateway reads the raw body itself — both for the JSON
  verbs and for the proxy/explicit-key `PUT` upload — so a parser that runs
  first leaves it empty: JSON verbs fail and uploads break. Mount the gateway
  **before** any global body parser (as above), or scope the parser so it skips
  this route.
</Callout>

## Any other runtime

The core accepts any consumer that can give it a Web `Request`. If your framework isn't listed, skip the binding and call `router.handle(request)` directly from any handler that exposes a `Request` (or constructs one):

```ts
const response = await router.handle(request);
```
